# Home Hero Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the above-the-fold hero of the Riverside Luxury Cruises marketing home page, replace `app/page.tsx` with it, and move the existing LiveKit `<App />` to `/agent`.

**Architecture:** A Next.js Server Component `Hero` composes three feature components (`TopUtilityBar`, `PrimaryNav`, `FloatingActions`) over a full-bleed `<Image>` background. All non-primitive components live under `components/home/`. Primitives reused from `components/ui/` (Button, Separator). Icons from `@phosphor-icons/react`. Source Serif 4 added as `--font-serif` for the title; Inter (already in layout) handles UI.

**Tech Stack:** Next.js 15 (App Router, Turbopack), React 19, TypeScript, Tailwind CSS v4, `@phosphor-icons/react`, `next/font` (Inter + Source Serif 4), `next/image`, `next/link`. No test framework is configured in this repo, so verification is via `pnpm lint`, `pnpm build`, and visual check at `pnpm dev`.

**Spec:** [`docs/superpowers/specs/2026-05-14-home-hero-design.md`](../specs/2026-05-14-home-hero-design.md)

## Notes for the Implementer

- **No test framework is configured.** Visual + build verification is what we have. Do not introduce one as part of this plan.
- **Cream / green colors are inlined hex** (`#EFE9DF`, `#5C6E5C`). Tune these visually against the mockup if they don't feel right; do not promote to Tailwind theme tokens in this iteration.
- **Logo overflow** (the seahorse hanging below the cream bar) is achieved by absolutely positioning the logo at `top-0` of the `PrimaryNav` and using a height (~110-130px) larger than the bar height (80px). Adjust the logo height if the visual doesn't match.
- **Phosphor icons** are imported from `@phosphor-icons/react/dist/ssr` so they're tree-shaken in Server Components.
- **Image priority:** the hero image uses `priority` because it's LCP. The logo also uses `priority` because it's above the fold.
- **`overflow-x-hidden`** is already applied on `<body>` (see `app/layout.tsx`), so absolute-positioned floating actions won't introduce horizontal scroll.
